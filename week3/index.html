Chapter 5 (Objects)

  • Objects can have a lot of info in them. We can access this info super easy.
  • You can call a function with an object like this: objectName.function();
  • You can check if a property exists in an object with hasOwnProperty()
  • Object.values() and object.entries() give you the values and key-value pairs.
  • Adding a property is easy, just type it in! Ex: object.newpropertyname = "nameOfKey-ValuePair";
  • You can change a property value by setting it equal to another key-value pair
  • You can delete a key-value pair by doing this: delete objectName.propertyName
  • Objects can be nested. You can have an object within an object.
  • Obejcts can be a parameter in a function!

    • DOM - Document Obejct Model

  • This turns everything on a webpage into a node. Why? So its easier to grab info. I think
  • So all of the "document." has to do with utilizing the DOM
  • Document.body returns the body element of a web page, as we saw in the previous example.
  • Document.links returns a node list of all the elements and elements that have an href attribute.
  • Document.anchors returns a node list of all the elements that have a name attribute.
  • Document.forms returns a node list of all the forms in the document. This will be used when we cover forms in Chapter 8.
  • innerHTML - returns all the child elements of a element as a string of HTML.
  • You can change the CSS styles of something too through JS.

    • Events - Chapter 7

  • addEventListener() is used on a node, has the code listen until the user does something, and then calls to action some funtion or code. For example: document.body.addEventListener('click', doSomething); where doSomething is a function.
  • You don't have to assign it to a node, like above. You could assign it to the entire page like this: addEventListener('click', () => alert('You Clicked!'));
  • You could also create a function, and within that function, an event listener occurs.
